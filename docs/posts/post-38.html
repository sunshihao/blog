    2024 碎碎念

    2025-01-05
    6 分钟
    年度总结

    大家好，我是 luckySnail。 写这篇文章的时候已经是 2025 年的第五天了，站在这个时间点，回顾过去一年，虽然有不足，遗憾。但是总体来讲还是向前的。写习惯了技术文章，可以一气呵成，突然写没有主题的文章，我一时间看着白纸不知道说什么，那就想到什么就说什么吧！

    AI 对我的改变

    过去这一年中，我打字最多的地方不是代码编辑器，也不是微信输入框，而是 AI 对话输入框。已经真的成为了一个严重依赖 AI 的开发者。 在过去的一年中，我写关于 AI 的文章应该不低于 10 篇，AI coding 在我看来已经非常强大，这一切都得益于 claude 3.5 sonnet 和 cursor，如果你还不了解，可以看我的文章: https://www.luckysnail.cn/posts/post-27 。如果你想要深入关注这个领域，我推荐你关注 「宝玉」，他的个人博客： https://baoyu.io/ 。 由于现在模型已经非常强大，我觉得无需非常专业的 prompt 我们都可以得到不错的生成的结果，关键是需要不断尝试，寻找经验，这里有一些我已经整理好的经验： https://www.luckysnail.cn/posts/post-31 ，宝玉也分享了自己的经验： https://baoyu.io/blog/2025-prompt-engineering-tips 。 在这一年中，我使用过的 AI 工具有：

    • 阿里的通义灵码、通义千问: https://tongyi.aliyun.com/
    • 腾讯的 VSCode 编程助手: https://copilot.tencent.com/
    • 更懂程序员的 devv.ai: https://devv.ai/
    • github copilot : https://github.com/features/copilot
    • deepseek: https://chat.deepseek.com/
    • gmini : https://gemini.google.com/
    • cursor: https://www.cursor.com/
    • mascode: https://www.marscode.com/
    • 豆包: https://www.doubao.com/chat/

    以上工具中目前在使用的是 devv 和 github copilot，devv 聚合了基本上最强的大模型，并且在 RGA 和交互方面我觉得是很好的，github copilot 目前是结合 VSCode 来使用，它目前会提供免费的额度，而且目前支持的能力也非常多，够用了。 AI 对程序员来说绝对是利好的，它让普通的我能够快速让想法落地；让普通的我快速学会一个新的知识；让普通的我看到超级个体的希望。 新的一年，我应该会更少的注意力在尝鲜新的 AI 工具，当然出圈的另说。

    认知

    2024 年，毫无疑问是我对自己认识更加清晰，深刻的发现自己认知不足的一年。这一年我浅薄的认识到了：

    • 开发完成，仅仅是开发，上线后的维护，回归测试等等才是重头戏
    • 代码 review 很重要：写完代码也仅仅说刚开始，还需要思考是否是最佳实现，是否有业务没有兼容的场景等等。
    • 技术仅仅是解决问题的手段
    • 好的设计可以减少后期和客户沟通，坏的设计全是反馈
    • 功能实现很重要，易用性也同样重要
    • 好的沟通能够减少很多工作
    • 关注数据，关注客户，关注业务价值
    • 世界是一个巨大的草台班子，人生是旷野
    • 热情是第一生产力，passion
    • 还有很多很多。。。 以上很多我都没有知行合一，其实我仅仅说短暂的知道，记录下来，新的一年我要带着更加谦卑平和的心态去生活，去工作

    注意力

    这一年，我分享了一个非常有名的症状：错失恐惧症（FOMO： Fear of Missing Out）。我发现我也是病入膏肓了。2024 是被无效信息绑架的一年，因为一些无效信息还徒增的烦恼和焦虑。

    这一年，我虽然很多时间在娱乐，但是同比之前的自己更加自律，新的一年只要比 2024 更好一点我就满足了

    这一年，除了看无用信息和娱乐，我也是有在学习新的知识的，

    希望新的一年更多关注自身，更加松弛一点。

    希望新的一年除了年龄 + 1 ，在其他方面也能加加加。

    学习

    这一年，我这一个抠抠搜搜的人也为知识付费了大几千块，主要在

    • 编程导航和前端垂直领域的知识星球，个人来看此类更多是为社区付费
    • AI coding ，直接忍痛买了 devv 年费，我觉得是我 2024 最具性价比付费项目之一
    • 体系化学习教程，买了一些二手课程，也购买了一些掘金小册，此类我理解是别人花时间帮我整理好学习路线，我按照他的学习路线来渐进式学习 这一年，我更少依赖视频学习，觉得比较低效，尝试 AI + 官网（或者整理好的学习路线）来学习，学习效果目前来看更加好，视频教程有很大的缺点是：
    • 不确定性：学习过程中，视频作者可能会发散 ，导致学习额外的知识，额外的知识使用不到也就等于没学
    • 心里负担高：主要体现在一般学习教程时长比较长 这一年，我更多践行费曼学习法，在学习途中，我会把重要却容易忘记的知识专门输出技术文章来加深我的印象，即使后面忘记也可以找补。实际发生的就是我在学习完 docker 后，真的有一个需求是为项目添加 docker file 。我之前记录的文章帮了大忙 这一年，我也更加关注学习方法论，推荐两篇文章:
    1. https://mp.weixin.qq.com/s/6vAzb0Vl9-KzFG10ceBrSw ，
    2. http://blog.yuanjin.tech/article/103

    值得反复看，里面很多我都没有吸收转化为实际行动。 以上就是这一年在学习过程中的碎碎念。
